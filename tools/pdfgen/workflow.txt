# MultixAI — Complete Project Workflow

## Overview
MultixAI is a full-stack web app with a React client (Vite) and a Node/Express server. It provides AI-powered utilities (image generation, background removal, resume review, article writing, etc.) and integrates with Cloudinary for media and uses multer for uploads.

## Repo Structure
- client/: React (Vite) frontend
  - src/: components, pages, assets
  - public/: static files
  - package.json: client scripts and deps
- server/: Node/Express backend
  - controllers/: `aiController.js`, `userController.js`
  - routes/: `aiRoutes.js`, `userRoutes.js`
  - configs/: `cloudinary.js`, `db.js`, `multer.js`
  - middlewares/: `auth.js`
  - server.js: Express app entry

## Architecture & Data Flow
1. User interacts with React UI (client/src/pages).
2. Client makes HTTP requests to server endpoints in `routes/`.
3. `aiController` handles AI operations (calls external AI services or internal logic).
4. Uploaded images are processed via `multer` and optionally sent to Cloudinary (`configs/cloudinary.js`).
5. Responses are returned to the client and rendered by components.
6. Authentication is handled by `middlewares/auth.js` for protected endpoints.

## Development Setup (Local)
1. Install dependencies:
   - Frontend: `cd client && npm install`
   - Backend: `cd server && npm install`
2. Environment variables: create `.env` files as needed (Cloudinary keys, DB URI, JWT secret).
3. Start services:
   - Backend: `node server/server.js` (or use `nodemon`)
   - Frontend: `cd client && npm run dev` (Vite dev server)
4. Open browser at the Vite dev URL (usually `http://localhost:5173`).

## Backend Details
- `server.js`: sets up Express, connects DB (`configs/db.js`), mounts routes from `routes/`.
- `routes/aiRoutes.js`: endpoints for image generation, removal, etc.
- `controllers/aiController.js`: orchestrates AI calls and Cloudinary uploads.
- `middlewares/auth.js`: checks JWT tokens and protects user-specific routes.
- `configs/multer.js`: handles multipart uploads.

## Frontend Details
- `src/pages`: each page corresponds to a feature (GenerateImages, RemoveBackground, ReviewResume, WriteArticle, etc.).
- `src/components`: reusable UI (Navbar, Sidebar, Plan, Hero, Footer).
- `src/App.jsx`: routes and layout wiring for the SPA.
- `assets/assets.js`: static asset references.

## Common Workflows
- Image generation:
  1. User submits prompt on UI.
  2. Client POSTs to `/api/ai/generate-image`.
  3. Server `aiController` calls AI service, receives image, uploads to Cloudinary, returns URL.
  4. Client displays returned image.

- Remove background:
  1. User uploads image via client.
  2. File arrives at server via `multer`.
  3. Server calls background-removal AI or Cloudinary transform.
  4. Returns processed image URL.

- Authentication flow:
  1. User logs in/registers via client forms.
  2. Server validates credentials (`userController`), returns a JWT.
  3. Client stores token (preferably in memory or secure storage) and includes it in Authorization headers for protected endpoints.

## Testing & Debugging
- Use browser devtools for frontend issues.
- Use Postman or curl to test server endpoints.
- Add console logs in controllers for traceability; prefer structured logging in production.

## Build & Deploy
- Frontend build: `cd client && npm run build` → produces static assets.
- Backend: deploy Node server to host supporting Node (e.g., Vercel Serverless functions, Heroku, DigitalOcean App Platform).
- Static assets can be served by the backend or deployed separately (e.g., Vercel for frontend, server on another host).
- Ensure environment variables (Cloudinary, DB, JWT secret) are configured in the deployment environment.

## CI / CD Recommendations
- Run lint and tests on PRs for client and server.
- Build frontend and run smoke tests.
- Deploy backend and frontend artifacts using environment-specific variables.

## Security & Best Practices
- Never commit secrets; use environment variables.
- Validate and sanitize user uploads.
- Limit file sizes in `multer` config.
- Use HTTPS and set secure cookie flags if using cookies.
